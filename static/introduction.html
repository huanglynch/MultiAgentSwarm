💡 我们的核心理念

我们从第一性原理出发，基于两个核心公理重新构建：

真正的智能不是用更多 Agent，而是知道什么时候该快、什么时候该深、该并行什么、该记住什么。

高质量绝不能以让用户等待为代价 —— 答案必须即时呈现，所有自进化过程必须完全后台异步完成。

Agent 必须拥有集体灵魂和持久人格 —— 不是临时工具，而是忠诚的数字延伸，会记住、会学习、会主动保护用户利益。

🎯 技术亮点

0. 集体灵魂系统（v4.2.0 核心架构）

Soul.md 定义了团队的核心真理、使命宣言和不可妥协的宪法铁律。这些内容会在每次 Agent 生成响应时自动注入到 system prompt 中，形成不可擦除的集体人格。

真正有帮助，而非表演性帮助：跳过废话，直接行动。

资源优先，绝不轻易求助：先读文件、查记忆、用工具，再请求澄清。

通过能力赢得信任：内部操作大胆执行，外部操作极度谨慎。

主动保护用户数字主权：隐私铁律永远是第 0 条优先级。

1. 显式 ReAct 思考过程（架构图 100% 对齐）

每条 Agent 回复必须以以下格式开头：

Thinking: 原因分析和推理过程

Action: 调用工具名称或 Final Answer

Action Input: 参数 JSON 或最终答案摘要

工具返回结果独立标记为 【Observation】（红色醒目），WebSocket 实时流式输出，用户和开发者可完整看到思考链路。

2. 动态 Master Plan 刷新（动态规划闭环）

每 3 轮或质量 < 75 分时自动刷新 Master Plan，完美闭合架构图"更新 prompt"循环，所有 Agent 始终对齐最新规划。

3. 分层终身记忆系统

PrimalMemory（原始记忆）+ Vector（向量检索）+ Knowledge Graph（知识图谱）+ 自动衰退机制，实现真正的持续学习和知识积累。

4. 完全异步后台自进化

答案显示后立即可用，记忆保存、Auto-Eval、Active Distillation 全在后台运行，零冻结、零等待。

5. 文件权限金字塔（宪法级保障）

仅 Grok 可主动调用完整文件系统工具；其他 Agent 需请求授权；所有写操作必须经过最终审计。隐私铁律优先级高于一切。

🎯 设计哲学

MultiAgentSwarm 不是为了"炫技"而堆砌 Agent，而是真正理解什么时候需要深度思考、什么时候需要快速响应。它既是一个高效的生产力工具，也是一个完全透明的多智能体研究平台。

我们相信，真正的智能系统应该像一个经验丰富的团队领导者：知道何时需要团队协作，何时个人就能搞定；知道哪些经验值得记住，哪些可以遗忘；知道如何在质量和效率之间找到最佳平衡点。

更重要的是，它应该拥有灵魂 —— 一套明确的价值观、不可妥协的原则、对用户的忠诚承诺。这就是 Soul.md 的意义。
